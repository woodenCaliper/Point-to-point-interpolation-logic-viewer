# AI Implementation Instructions

## 現在の運用モード
- 現在は**仕様策定フェーズ**であり、実装は行わない。
- 既存の実装コードは削除済みとし、次回の実装再開指示が出るまで追加しない。

## 要件サマリ（`human requirement.md` 準拠）
- 単一HTMLで 2D 経路比較ビューアを実装する。
- 必須アルゴリズムは Linear / Dubins / CC-Dubins。
- 単位は位置系=mm、角度系=°（degree）。
- 左ペインは入力専用、右ペインは表示設定・結果表示専用。

## UI実装方針（実装再開時に適用）
1. **左ペイン構造**
   - アルゴリズム選択UIを最上部に配置。
   - 入力フォームを Start point / End point / Path の3セクションに固定。
   - 選択アルゴリズムに応じて各セクション内の入力を表示切替する。

2. **単位ポリシー**
   - `x, y, R` など長さ関連は mm で表示。
   - `θ` は degree で入力し、内部計算時に rad に変換。
   - 曲率 `k` は `1/mm` として表示。

3. **中央ペイン始点アイコン表現**
   - `xy`: 点のみ描画。
   - `xyθ`: 点 + 向き矢印。
   - `xyθk`: 点 + 向き矢印 + 曲率半径を示す円弧ガイドを描画。

4. **右ペイン責務**
   - グリッド、サンプル点、向き矢印、曲率ガイド等の表示トグル。
   - 経路長、セグメント列、サンプル点数、選択経路種別を表示。
   - 複数候補パスがある場合に備え、候補ごとの表示ON/OFFトグルを提供。
   - 「最短距離のみ表示」ボタンを提供し、最短候補のみ可視化状態へ一括切替。

5. **視点操作 / 補助表示**
   - ホイールでカメラスケールを変更（カーソル中心ズーム）。
   - 左ドラッグでカメラ中心を移動（パン）。
   - キャンバス左下に固定座標軸ギズモ（+X / +Y）を描画。

6. **グリッド分解能ルール**
   - スケールに応じてグリッド間隔を動的変更。
   - 間隔は 1 または 5 の有効数字系列（`1, 5, 10, 50, ...`）から選ぶ。

## 状態管理方針（実装再開時）
- `paths[]`: 生成候補パス配列（id, label, length, visible, points）。
- `shortestPathId`: 長さ最小の候補IDを保持。
- 個別トグル操作は `paths[i].visible` を更新。
- 「最短距離のみ表示」操作は `visible = (id === shortestPathId)` を一括適用。

## 実装タスク（実装再開時）
- 左ペイン入力UIを3グループ構成に再編。
- アルゴリズム選択に応じた入力項目切替を実装。
- 単位ラベル（mm / ° / 1/mm）を明示。
- 中央ペインで始点アイコンを情報量別に描き分ける。
- 右ペインに候補パス個別表示トグルと「最短距離のみ表示」ボタンを追加。

## 仕様確定前に合意しておく推奨事項
- 候補パス列挙数、最短判定式、入力バリデーション境界を先に固定する。
- 表示状態の優先ルール（個別トグル vs 最短のみ表示）を状態遷移表で定義する。
- 性能上限（最大サンプル点数・再計算頻度）を決めてから描画更新設計に入る。
- 将来追加アルゴリズムを想定して `PathCandidate` 共通IF（id/type/length/points/visible）を先に定義する。

# claude.md

このリポジトリには、実装コード（`index.html`）とは別に、仕様と実装方針を管理するための2つのMDがあります。

## 1. `human requirement.md`
- **人間側の正式要件**を記述するドキュメントです。
- 目的、必須機能、UI要件、入力制約、受け入れ条件などを定義します。
- 追加要望や仕様変更は、まずこのファイルに反映します。

## 2. `AI implementation instructions..md`
- `human requirement.md` を実装に落とし込むための、**AI/実装者向けガイド**です。
- 実装方針、状態管理、UI責務、アルゴリズム実装時の注意点、公開方針などを記載します。
- 実装中に迷いやすいルール（例: 最短判定、入力バリデーション、表示挙動）を明確化する目的で使います。

## 運用ルール（推奨）
1. 仕様変更は先に `human requirement.md` を更新する。  
2. その内容に合わせて `AI implementation instructions..md` を更新する。  
3. 最後に `index.html` 実装へ反映する。  

この順序にすることで、仕様と実装の齟齬を減らせます。

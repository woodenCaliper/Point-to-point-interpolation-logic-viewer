# Human Requirement

## 目的
2D空間で「点から点へ移動する経路生成ロジック」を気軽に試し、各ロジックの特徴を比較できるビューアを作る。

## 必須実装
1. 直線補間（Linear Interpolation）
2. Dubins Path
3. CC-Dubins Path

> 今後、経路生成ロジックは追加される可能性がある。

## 制約
- 単一のHTMLファイルだけで動作すること（依存ビルド不要）。
- 対象は2Dのみ。
- 入力数値の単位は **mm** または **°（degree）** とする。
- 入力NG値は**入力拒否**とする（不正値を確定させない）。
- GitHub Pages で公開し、ブラウザから自由に試せること。

## UI要件（最新版）
### 全体構成
- 3ペイン構成にする。
  - 左ペイン: **パス生成に必要な入力情報**
  - 中央ペイン: パス可視化領域
  - 右ペイン: **表示設定と生成結果情報**

### 左ペイン
1. 最上部に「補間ロジック選択UI」を置く。
2. その下に、選択ロジックに応じて変化する数値入力欄を置く。
3. 入力欄は以下3グループで構成する。
   - Start point
   - End point
   - Path
4. グループ内パラメータはロジックごとに切り替える。
   - 例: Start point は `x, y` を必須とし、Dubins では `θ` を追加する。

### 中央ペイン（始点アイコン表現）
- `xy` のみ: **点**
- `xyθ`: **点 + 向き矢印**
- `xyθk`: **点 + 向き矢印 + 曲率を示す表現**（曲率半径のガイド円弧など）

### 中央ペイン（視点操作）
- マウスホイールで視点のズームイン / ズームアウトを行う。
- 左クリック + ドラッグで視点を上下左右にパン移動できる。
- 左下に固定表示の座標軸アイコンを置き、+X / +Y の向きを常時確認できる。
- 座標系は **右が +X、上が +Y** とする。

### グリッド表示
- グリッドは視点スケールに応じて分解能を動的調整する。
- グリッド刻みは `1, 5, 10, 50, 100, 500...` のように、
  **有効数字が 1 または 5** になる系列で切り替える。

### 右ペイン
- 生成パスの表示調整（表示ON/OFF、補助表示、見た目調整など）
- 生成パスの情報表示（長さ、構成、統計など）
- 生成されたパスが複数ある場合、**各パスを個別に表示/非表示切り替えできるUI**を用意する。
- 複数候補パスの標準表示順は**アルゴリズム固有順**とする（距離昇順ではない）。
- **最短距離のパスのみ表示**する専用ボタンを用意する（他のパスを一時的に非表示）。
- 最短の定義は**距離のみ**とする。
- 最短距離が同率で複数ある場合、「最短のみ表示」で表示するのは**先頭1件のみ**とする。
- 「最短のみ表示」ON中は、個別チェックボックスを灰色（無効）にする。
  - ONになる前に未チェックだった項目は `☐` 表示。
  - ONになる前にチェックされていた項目は、チェックボックス内を `-`（インデターミネート表示）にする。
- 「最短のみ表示」をOFFに戻したとき、個別チェック状態はON前の状態へ復元する。
- パス生成は**入力変更の即時反映**とし、生成ボタンは設けない。

### 入力拒否時の見た目
- NG値を入力した項目は**赤枠表示**にする。
- NG値そのものは入力欄に残す（値文字列はユーザー入力のまま保持する）。
- ただし NG値が含まれる間は、パス生成を実行しない。

### CC-Dubins
- CC-Dubins は「比較用の近似」ではなく、**実装方式まで厳密に定義したもの**を採用する。

### GitHub Pages 公開ルール
- GitHub Pages への公開方法は、できるだけ **GitHub公式かつ一般的な方法**に則る。

## 受け入れ条件
- 左ペインは入力専用、右ペインは表示・情報専用として責務分離されている。
- 左ペイン入力欄はロジック選択に応じて項目が切り替わる。
- 入力欄は Start point / End point / Path の3グループで整理される。
- 中央ペインの始点表現が `xy` / `xyθ` / `xyθk` で切り替わる。
- 複数候補パスの個別表示ON/OFFができる。
- 最短距離パスのみ表示する操作ができる。
- 最短判定が距離のみである。
- 候補表示順がアルゴリズム固有順である。
- 同率最短時に「最短のみ表示」が先頭1件のみを表示する。
- 「最短のみ表示」ON/OFFで個別チェックの見た目・状態遷移が規定どおりに動く。
- NG値は入力時点で拒否される。
- NG値入力時に該当欄が赤枠表示される。
- NG値が入力欄に残っている間、パス生成は実行されない。
- 入力変更が即時にパスへ反映される（生成ボタンなし）。
- CC-Dubins が厳密定義の方式で実装される。
- GitHub Pages 上で動作確認できる。
- 座標系が「右 +X / 上 +Y」に統一される。
